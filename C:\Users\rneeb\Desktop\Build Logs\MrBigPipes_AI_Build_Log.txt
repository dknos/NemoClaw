================================================================================

TIME FRAME:
  14:15 UTC - Began disabling auto-posting feature (security fix)
  14:20 UTC - Edited discord-bridge.js to comment out auto-post logic
  14:21 UTC - Attempted to restart Pipes
  14:21 UTC - PROCESS CRASHED with SyntaxError: Missing catch or finally after try
  14:22 UTC - Root cause identified
  14:23 UTC - Fix applied: removed orphaned code
  14:24 UTC - Pipes restarted successfully
  DOWNTIME: 3 minutes

SEVERITY: HIGH (agent offline, blocking user access)

ROOT CAUSE:
  When disabling the auto-post-to-Instagram feature, I commented out the main
  `if` statement but left dangling error handling code (catch/finally blocks).

  BROKEN CODE:
    // DISABLED: No auto-posting...
    // const userWantsPost = /pattern/...
    // if (userWantsPost && !bufferMatch && ...) {
        await msg.reply(`Auto-post failed: ${e.message}`);  // ← orphaned
      }  // ← orphaned closing brace
    }

  The JavaScript parser cannot match these closing braces to any opening block.

FIX APPLIED:
  Removed all orphaned code completely. Now:

    // DISABLED: No auto-posting to Instagram without explicit user request
    // User must explicitly run /post command or ask the agent to post

PREVENTION:
  Added `make validate-agents` Makefile target
  Run this BEFORE any pm2 restart:

    make validate-agents          # Syntax validation (< 1 second)
    npm run lint:cli              # Type checking (< 2 seconds)
    pm2 restart discord-bridge    # Deploy
    pm2 logs discord-bridge --lines 10  # Verify startup
