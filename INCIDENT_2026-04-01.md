# Incident Report: Pipes Offline (2026-04-01)

## Summary
Pipes (mrbigpipes_ai / discord-bridge) crashed on startup due to a syntax error introduced during code editing. Agent was offline for ~3 minutes. Root cause: incomplete code removal left orphaned closing braces.

## Timeline
- **14:15 UTC** - Began disabling auto-posting feature to prevent unauthorized Instagram posts
- **14:20 UTC** - Edited discord-bridge.js to comment out auto-post logic
- **14:21 UTC** - Attempted to restart Pipes
- **14:21 UTC** - Process crashed with `SyntaxError: Missing catch or finally after try`
- **14:22 UTC** - Identified syntax error in code
- **14:23 UTC** - Fixed: removed orphaned error handling code
- **14:24 UTC** - Pipes restarted successfully

## Root Cause

When disabling auto-posting, the edit removed the `if` statement but left dangling code:

```javascript
// BEFORE (broken)
// DISABLED: No auto-posting...
// const userWantsPost = /pattern/...
// if (userWantsPost && !bufferMatch && ...) {
    await msg.reply(`Auto-post failed: ${e.message}`);  // ← orphaned (no try block)
  }  // ← orphaned closing brace
}
```

The JavaScript parser sees a `}` without a matching `{` after the commented `if`.

## Fix Applied

```javascript
// AFTER (fixed)
// DISABLED: No auto-posting to Instagram without explicit user request
// User must explicitly run /post command or ask the agent to post
```

Removed all orphaned code entirely.

## Prevention

### Immediate Actions
1. Added `make validate-agents` target to Makefile
2. Documented validation checklist in MEMORY.md
3. Created BUILD_LOG.md to track incidents

### Deployment Procedure (Going Forward)

Before restarting ANY agent:
```bash
make validate-agents          # Syntax check (< 1 second)
npm run lint:cli              # Type check (< 2 seconds)
pm2 restart discord-bridge    # Deploy
pm2 logs discord-bridge --lines 10 --nostream  # Verify
```

### Git Hooks
- `pre-push` hook runs type-checking (`tsc --noEmit`)
- This won't catch syntax errors in commented code blocks, but catches most issues
- **Always run `make validate-agents` before `pm2 restart`**

## Lessons Learned

1. **Comment-out != delete:** Partial edits can leave syntactically invalid code
2. **Validate before deploy:** The cost of `make validate-agents` (1 sec) << downtime (3 min)
3. **Error handling:** Don't leave orphaned catch/finally blocks
4. **Document incidents:** This prevents repeated mistakes
